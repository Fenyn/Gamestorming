# Mythos

Lane combat card game with city building. Built in Godot 4.6.

## Architecture

- **EventBus** autoload: all inter-system communication via signals
- **CardDatabase** autoload: loads .tres card resources, provides deck creation
- **GameState** autoload: authoritative game state, action validation
- **TurnManager** autoload: phase sequencing (Draw → City → Spells → Battle → Summon → Build)
- **CombatManager** autoload: lane-by-lane battle resolution
- **SpellManager** autoload: spell track advancement and effect resolution
- **NetworkManager** autoload: ENet P2P connection (dual simulation lockstep planned)

## Card Data

Card resources live in `resources/cards/nordic/`. Three types:
- `UnitData` extends `CardData`: attack, health, keywords
- `BuildingData` extends `CardData`: health, resource_generation, effects
- `SpellData` extends `CardData`: start_position, target_mode, triggers

Runtime mutable state uses `UnitInstance`, `BuildingInstance`, `SpellInstance` (RefCounted).

## Game Rules

- 30-card deck (units + buildings only). Spells are always-available side pool.
- Grand Lodge (HQ) starts in play at city center. 50 HP. Win by destroying opponent's HQ.
- Resources accumulate (bank). Generated by Grand Lodge + Mead Halls during City Effects.
- Hand limit: 7 cards. Second player draws 6 cards (first player draws 5).
- Spell track: 5 positions, spells count down to 0 then resolve.

## Code Style

- Strongly typed GDScript: explicit type annotations, `class_name`, `@export`
- No `:=` with untyped sources
- Procedural scene construction (board slots built in code)
- Signals via EventBus for loose coupling
